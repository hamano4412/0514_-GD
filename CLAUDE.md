# 新卒採用GD評価アプリ

新卒採用のグループディスカッション（GD）をZoomブレイクアウトルームで実施した録画データを取り込み、AIが各学生の発言を解析して評価軸ごとにスコア化・根拠提示する非同期評価支援アプリ。

## プロダクトの目的

現場社員がGDに同席せずとも、後から短時間で公平・客観的な評価ができる状態をつくる。40分の動画を全部見る必要をなくし、AIが抽出したハイライトの数分間だけで事実確認できることを目指す。

## 主要機能

### 1. データの取得と構造化
- Zoomブレイクアウトルームの録画データ（動画＋音声）をアプリに取り込む
- 音声をテキスト化（文字起こし）
- 学生A〜Dそれぞれの発言録として話者分離・構造化
- 発言にはタイムスタンプを付与し、後段の動画再生リンクに利用する

### 2. AIによる一次解析とファクトの抽出
文字起こしテキストと音声データに対し、AIが自社の評価軸に基づくプロンプトで解析する。

評価軸は以下の3つ:

- **主体性**: 議論の口火を切った回数、沈黙を破った回数、全体の総発話量などをカウントし、積極性を示す発言を抽出
- **リーダーシップ**: 発言が少ない参加者へ意見を求めた箇所、議論が発散した際に「要するにこういうことですね」とまとめた発言を抽出
- **認知能力**: 結論ファーストで話しているか、論理の飛躍がないか、お題に対して的確な仮説を立てている発言をテキストから読み取る

### 3. 評価者用ダッシュボード（非同期確認）
現場社員が後からアクセスする管理画面。

- **個人別サマリー**: 学生ごとに「主体性・リーダーシップ・認知能力」のAIスコア予測と、その根拠となる発言の要約を表示
- **タイムスタンプ・ハイライト機能**: AIが「評価の決め手になる」と判断した重要発言（例：「Aさんの認知能力の高さがわかる発言」）に動画再生リンク（タイムスタンプジャンプ）を付与
- 社員は40分の動画を全部見る必要がなく、ハイライト数分間で事実確認ができる

## 評価軸サマリ

| 評価軸 | 抽出するシグナル例 |
| --- | --- |
| 主体性 | 口火を切った回数 / 沈黙を破った回数 / 総発話量 / 積極性を示す発言 |
| リーダーシップ | 発言が少ない人への意見promote / 議論の要約・収束発言 |
| 認知能力 | 結論ファースト / 論理の飛躍の有無 / 仮説の的確さ |

## 設計上の重要原則

- **AIの出力は必ず根拠（発言の引用＋タイムスタンプ）と一緒に提示する**。スコアだけを出して評価者がブラックボックスを信じる構造にしない
- **評価者の最終判断を代替しない**。AIは「ファクト抽出」「ハイライト提示」までを担い、合否判断は人間が行う
- **話者分離の精度がプロダクト全体の品質を左右する**。学生A〜Dの取り違えは評価結果の信頼を即座に毀損するため、ここの精度確保を最優先する
- **非同期前提のUX**。社員はGDに同席していないので、文脈が分からなくても短時間で評価できるダッシュボード設計にする

## 想定ユーザー

- **評価者（現場社員）**: GDに同席せず、ダッシュボードで非同期に評価する
- **採用担当**: 評価結果を集計し、合否判断や応募者へのフィードバックに使う
- **応募者（学生A〜D）**: 直接アプリは触らないが、録画・解析対象として最も配慮すべき対象
